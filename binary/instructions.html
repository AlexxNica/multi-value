Instructions¶

Instructions are encoded by opcodes. Each opcode is represented by a single byte, and is followed by the instruction’s immediate arguments, where present. The only exception are structured control instructions, which consist of several opcodes bracketing their nested instruction sequences.

Note

Gaps in the byte code ranges for encoding instructions are reserved for future extensions.

Control Instructions¶

Control instructions have varying encodings. For structured instructions, the instruction sequences forming nested blocks are terminated with explicit opcodes for \(\href{../syntax/instructions.html#syntax-instr-control}{\mathsf{end}}\) and \(\href{../syntax/instructions.html#syntax-instr-control}{\mathsf{else}}\).

Block types are encoded in special compressed form, by either the byte \(\def\mathdef476#1{\mathtt{0x#1}}\mathdef476{40}\) indicating the empty type, as a single value type, or as a type index encoded as a positive signed integer.

\[\begin{split}\begin{array}{llclll} \def\mathdef418#1{{}}\mathdef418{block type} & \href{../binary/instructions.html#binary-blocktype}{\mathtt{blocktype}} &::=& \def\mathdef477#1{\mathtt{0x#1}}\mathdef477{40} &\Rightarrow& \epsilon \\ &&|& t{:}\href{../binary/types.html#binary-valtype}{\mathtt{valtype}} &\Rightarrow& t \\ &&|& \href{../syntax/instructions.html#syntax-instr-control}{\mathsf{unreachable}} \\ &&|& \def\mathdef479#1{\mathtt{0x#1}}\mathdef479{01} &\Rightarrow& Note

The \(\href{../syntax/instructions.html#syntax-instr-control}{\mathsf{else}}\) opcode \(\def\mathdef495#1{\mathtt{0x#1}}\mathdef495{05}\) in the encoding of an \(\href{../syntax/instructions.html#syntax-instr-control}{\mathsf{if}}\) instruction can be omitted if the following instruction sequence is empty.

Unlike any other occurrence, the type index in a block type is encoded as a positive signed integer, so that its LEB128 bit pattern cannot collide with the encoding of value types or the special code \(\def\mathdef496#1{\mathtt{0x#1}}\mathdef496{40}\), which correspond to the LEB128 encoding of negative integers. To avoid any loss in the range of allowed indices, it is treated as a 33 bit signed integer.

Parametric Instructions¶

Parametric instructions are represented by single byte codes.

\[\begin{split}\begin{array}{llclll} \def\mathdef418#1{{}}\mathdef418{instruction} & \href{../binary/instructions.html#binary-instr}{\mathtt{instr}} &::=& \dots \\ &&|& \def\mathdef497#1{\mathtt{0x#1}}\mathdef497{1A} &\Rightarrow& \href{../syntax/instructions.html#syntax-instr-parametric}{\mathsf{drop}} \\ &&|& \def\mathdef498#1{\mathtt{0x#1}}\mathdef498{1B} &\Rightarrow& \href{../syntax/instructions.html#syntax-instr-parametric}{\mathsf{select}} \\ \end{array}\end{split}\]
